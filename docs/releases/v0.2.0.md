# Codex-Skin v0.2.0

Codex-Skin v0.2.0 引入面向主题商店的增量分发链路，减少目录刷新和主题浏览产生的重复下载。

## 新功能

- 使用轻量 `desktop-catalog-v2.json` 远程目录，不再下载完整 Store 仓库 ZIP；
- 使用 HTTP ETag 条件请求，目录没有变化时不重复传输；
- 主题预览进入可视区域时才按需下载；
- 应用主题前才下载对应清单、背景、Logo 和宠物资源；
- 根据路径、精确大小和 SHA-256 识别缺失或变化的资源，实现主题级增量更新；
- 资源通过图片解码与完整主题校验后原子写入本地持久缓存；
- 远程目录下架的主题不再作为在线主题显示，已安装的签名版本仍可离线使用；
- GitHub、GH Proxy 和 GHFast 下载线路继续支持自动回退。

## 发布链路

- Codex-Skin-Store 自动生成内容寻址的桌面目录；
- Store 的发布复验已统一使用 Rust `dreamskin-verify`，移除旧 .NET 验证器；
- `.dreamskin` 的 SHA-256、Ed25519/RFC8785 签名导入与本地不可变版本库保持兼容。

Codex-Skin v0.2.0 introduces incremental theme delivery to avoid repeatedly downloading the complete Store catalog and assets.

## Highlights

- Lightweight `desktop-catalog-v2.json` synchronization instead of downloading the complete Store repository ZIP;
- HTTP ETag requests that transfer nothing when the catalog is unchanged;
- Lazy preview downloads as theme cards enter the viewport;
- Manifest, background, logo, and pet downloads only before a theme is applied;
- Resource-level incremental updates based on path, exact size, and SHA-256;
- Image decoding and full theme validation before atomic writes to the persistent local cache;
- Removed online themes disappear from the remote catalog while installed signed versions remain available offline;
- Automatic fallback across GitHub, GH Proxy, and GHFast remains available.

The Store publication verifier now uses the same Rust `dreamskin-verify` implementation as the client. Existing SHA-256 and Ed25519/RFC8785 `.dreamskin` imports remain compatible.

> The packages are not commercially code-signed, and the macOS PKGs are not notarized. Download only from this Release and verify the SHA-256 checksum before running.
